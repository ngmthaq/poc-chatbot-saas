## Approach Summary

- Generalize the current company-specific tool into a standard web search tool. The schema changes from `{ company }` to `{ query }`, and the tool answers any external-information query.
- Hybrid provider strategy mirroring `get-stock-price.tool.ts`: an optional keyed provider (Tavily, LLM-optimized, via `TAVILY_API_KEY`) is tried first when configured; otherwise it falls back to keyless providers (DuckDuckGo Instant Answer → Wikipedia), and finally to a Google search URL the user can open.
- "Adapt with both LiveKit and deepagent" requires no adapter changes: extending `BaseTool` and registering in `toolRegistry` means `livekit.adapter.ts` and `langchain.adapter.ts` wrap it automatically.
- The old `searchBranchInformation` tool name is retired in favor of `webSearch`.

## Functional Requirements

- New tool `WebSearchTool` with name `webSearch`, schema `{ query: string }`, returning a concise text result.
- Provider order: Tavily (only if `TAVILY_API_KEY` set) → DuckDuckGo Instant Answer → Wikipedia → fallback Google search URL.
- Each provider is best-effort: a provider error is logged and the next is tried (same loop pattern as the current file).
- When no provider yields data, return a message containing an openable web-search URL.
- Tool is registered in `registry.ts` and exported from `index.ts`, replacing the old export.

## Non-Functional Requirements

- Use `fetchWithTimeout` and `dedent` from `../utils/index` (existing convention).
- All types/interfaces live in `src/types/web-search.d.ts` — no inline `type`/`interface` declarations.
- No `.js` extensions in relative imports.
- No new runtime dependencies (Tavily called via `fetch`, not its SDK). `TAVILY_API_KEY` is optional; tool works with zero config.
- Must pass `pnpm typecheck` and `pnpm lint` in `packages/harness`.

## Risks & Assumptions

- Assumption: Tavily is the chosen optional keyed provider (LLM-optimized). If a different provider is preferred later, only `web-search.tool.ts` changes.
- Assumption: No callers depend on the literal tool-name string `searchBranchInformation` outside the registry/index (verified via grep — only those two files reference it).
- Risk: DuckDuckGo Instant Answer / Wikipedia return nothing for many general queries; the fallback URL message mitigates this. Inherent to keyless search and accepted per the "Hybrid" decision.
- Risk: Keyless web search quality is limited until `TAVILY_API_KEY` is provided — documented in the tool description/comments.
