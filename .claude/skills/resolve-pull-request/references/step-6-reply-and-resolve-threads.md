# Step 6 — Reply to and Resolve Threads

Once party-mode's review has **accepted** the changes for a comment, push the branch, then reply to and resolve that comment's thread.

## Push the changes first

```bash
git push           # push the accepted commits to the PR head branch
```

Replies should reference work that is actually visible on the PR, so push before replying.

## Reply, then resolve — per accepted thread

For each `resolve` row whose change was accepted, post a short reply stating **what changed** (and the commit/file if helpful), then mark the thread resolved. For `reply-only` rows (questions), post the answer and resolve only if the thread truly needs no code change.

Do **not** add any AI/agent attribution to the reply — no "Generated by", "Resolved by Claude", "Co-Authored-By: Claude", or similar signature lines. Each reply contains only the substance.

## Using MCP

Use the MCP tool to reply to each thread by its ID and to mark it resolved.

## Using the CLI

**GitHub** — reply to the review comment, then resolve the thread via GraphQL (resolving needs the thread node ID from Step 3):

```bash
# Reply to an inline review comment (by its REST comment id)
gh api repos/{owner}/{repo}/pulls/{number}/comments/{comment_id}/replies \
  -f body="<what changed>"

# Resolve the thread (thread node id from the reviewThreads query in Step 3)
gh api graphql -f query='
  mutation($threadId:ID!){
    resolveReviewThread(input:{threadId:$threadId}){ thread{ isResolved } } }' \
  -F threadId=<thread-node-id>
```

**GitLab** — add a note to the discussion, then resolve it:

```bash
glab api projects/:id/merge_requests/<number>/discussions/<discussion_id>/notes \
  -f body="<what changed>"

glab api --method PUT \
  "projects/:id/merge_requests/<number>/discussions/<discussion_id>?resolved=true"
```

**Bitbucket** — use the installed CLI's reply / resolve commands (flags vary by tool). If the CLI cannot resolve a thread, post the reply and tell the user the thread must be resolved manually.

## Do not resolve when

- The change was not accepted by party-mode's review (`incomplete` / `blocked`).
- The comment was a question the user has not yet answered.
- You could not tie the change to that specific comment.

Leave those threads open and report them in Step 7.
