## Approach Summary

- `@livekit/agents@1.4.3` declares `zod@^3.25.76 || ^4.1.8` as a peer dependency. Because `agent-worker/package.json` never lists `zod`, yarn does not install it, the lockfile lacks the resolution, and the Debian-glibc image (cycle-1 fix) crashes at startup when `@livekit/agents/dist/beta/workflows/task_group.cjs` does `require("zod")`.
- Source code in `agent-worker/src/{main,agent,config,logger}.ts` does not import `zod` directly — the requirement is purely transitive — so the fix is a single declaration in `package.json` plus a lockfile refresh. No source edits, no Dockerfile changes (cycle-1 work is preserved).
- Pin `^4.1.8` per user override. The peer-dep range explicitly allows v4, and `task_group.cjs` only uses `z.object`, `z.array`, `z.enum`, which are stable across v3.25 and v4. The user has elected v4 (likely to match a shared version with sibling services).
