## Approach Summary

Install `zod` as a runtime dependency. Add a single-responsibility validator file at `src/validators/room.validator.ts` that exports: a Zod schema (`joinRoomSchema`), an inferred TypeScript type (`JoinRoomBody`), and an Express `RequestHandler` middleware (`validateJoinRoom`). The middleware calls `schema.safeParse(req.body)`, overwrites `req.body` with the coerced result on success, and calls `next(createHttpError(422, ...))` on failure.

`RoomService.join(body: JoinRoomBody)` uses `crypto.randomUUID()` to derive the room name, calls `livekitRoomUtil.createRoom` then `livekitTokenUtil.createAccessToken`, and returns `{ roomName, token }`. The generated JWT is never logged. `RoomController` holds a single arrow-method `join: RequestHandler`. The route registers `validateJoinRoom` before `responseHandler(roomController.join)`.

---

## Functional Requirements

1. `zod` installed as a runtime dependency of `apps/livekit-server`.
2. `src/validators/room.validator.ts` exports `joinRoomSchema`, `JoinRoomBody`, `validateJoinRoom`.
3. `src/services/room.service.ts` exports `RoomService` + `roomService` singleton with `async join(body: JoinRoomBody)`.
4. `src/controllers/room.controller.ts` exports `RoomController` + `roomController` singleton.
5. `src/routes/room.route.ts` registers `POST /` with `validateJoinRoom, responseHandler(roomController.join)`.
6. `src/routes/index.ts` mounts room router at `/rooms`.
7. `POST /rooms/join` with valid body returns HTTP 200 with `{ status, data: { roomName, token }, timestamp }`.
8. `POST /rooms/join` with invalid body returns HTTP 422.

---

## Non-Functional Requirements

- `verbatimModuleSyntax` compliant: `import type` for type-only imports.
- Class + singleton export pattern matches existing services/controllers.
- Arrow methods for controller handlers.
- No `console.*` calls that log token, identity, or secret-derived values.
- `createHttpError` from `http-errors` for all error cases.
- `randomUUID` from `node:crypto` — no extra package.
